## Design Context

### Users
Developers encountering these demos through two paths: (1) the 30 Days of Workflow DevKit social campaign on X/Twitter, where first impressions drive curiosity, and (2) as reference implementations when evaluating whether to adopt Vercel Workflow DevKit. In both cases, the user is a developer who wants to quickly understand a workflow pattern, see it running, and grasp the code — not read documentation.

### Brand Personality
**Clean, modern, approachable.** The Vercel design ethos — elegant engineering that feels inviting rather than intimidating. Every demo should feel like a polished product, not a tutorial or prototype.

### Aesthetic Direction
- **Theme**: Dark-mode only, built on the Geist design system
- **Palette**: Near-black backgrounds (#0a0a0a, #171717), high-contrast light text (#ededed), semantic status colors (green/amber/red/cyan/blue) used consistently for workflow state
- **Typography**: Geist Sans for UI, Geist Mono for code — clean, modern, no serif
- **Layout**: Centered max-w-4xl container, responsive two-pane code workbench, generous spacing
- **Code is the hero**: Syntax-highlighted code panes with real-time gutter marks and line highlighting are the centerpiece of every demo
- **References**: Vercel dashboard, v0.dev, Next.js docs — restrained dark interfaces where content breathes
- **Anti-references**: Generic docs/tutorials (boring walkthroughs with no identity), over-designed marketing pages (gradients and fluff obscuring the code), raw unstyled prototypes (functional but visually careless)

### Design Principles

1. **Code is the interface.** The workflow source code is the primary UI element, not a secondary detail. Design decisions serve code readability and execution visualization above all else.

2. **Show, don't explain.** Each demo should be immediately runnable and visually self-documenting through color-coded state transitions, gutter marks, and real-time SSE streaming — minimize prose.

3. **Consistent vocabulary.** Green means success, amber means retry, red means failure, cyan means processing, blue means accent/action. This color language is universal across all 48 demos. Never deviate.

4. **Restrained polish.** Every element should feel intentional and considered, but never decorative. No gradients, no shadows-for-shadows-sake, no animation without purpose. The Vercel ethos: subtract until only the essential remains.

5. **Campaign-worthy at a glance.** Each demo must look compelling in a screenshot or social embed. The visual structure (badge → title → interactive code panes → status visualization) should read clearly even at thumbnail scale.

### Accessibility
- WCAG AA compliance
- Semantic HTML with skip-to-content links
- Keyboard navigable interactive elements
- Focus-visible indicators on all interactive elements
- Sufficient contrast ratios on dark background (primary text ~16:1, secondary text ~7:1)
